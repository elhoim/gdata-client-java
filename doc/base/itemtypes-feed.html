Item Types Feed

This read-only feed provides a list of attributes associated with each of the Google Base item types. See Attributes and Queries for some code samples on how to access this feed through the Google Base client library.

Contents

  1. Feed URL
  2. Elements
  3. Usage example

Feed URL

You can request a list of all attributes for all items in a specific locale or just for a single item type.

Use this URL to get the entire list of attributes for a given locale: 

http://www.google.com/base/feeds/itemtypes/<locale>

Use this URL to get a list of all attributes for a single item type: 

http://www.google.com/base/feeds/itemtypes/<locale>[/<itemtype>]

locale limits the feed to return information for a single location. locale is a required parameter. Currently, the following locales are defined:

United States en_US
Great Britain en_GB
Germany de_DE

itemtype limits the feed to return detailed attribute data for a single item type. To find a list of valid itemtype values for a given locale, query the feed without an itemtype parameter first.

For example, this url provides a list of all attributes for the US locale: 

http://www.google.com/base/feeds/itemtypes/en_US

This URL obtains only those attributes defined for the housing item type:

http://www.google.com/base/feeds/itemtypes/en_US/housing

You can also run batch queries on the itemtypes feed.

Elements

Google Base uses the following elements in the itemtypes feed:

Element Meaning
gm:item_type The name of the item type
gm:attributes A list of all the attributes supported by Google Base to describe items of this type.
gm:attribute A single attribute specified by name and type.

Note that these elements use the gm: namespace. This namespace contains the metadata attributes that are used in the item types and attributes feeds. No other feeds use them and they are read-only, so users do not need to use this gm: namespace to input the attributes. The only impact on users is that you will see an extra line in the response, giving the location of the namespace, xmlns:gm ='http://base.google.com/ns-metadata/1.0.

There is a link in the response that points to the snippets feed where more data can be found about this item type. This allows you to write code that automatically takes you to the next level of detail. The link uses a category search, with " /-/" prefix to indicate that the query is to get all items with the Jobs item type. 

<link rel="related" type="application/atom+xml" href="http://www.google.com/base/feeds/snippets/-/jobs" title="Items in jobs"/>

Usage example

Request:

http://www.google.com/base/feeds/itemtypes/en_US/jobs

Response: 

<?xml version='1.0' encoding='UTF-8'?>
<entry xmlns='http://www.w3.org/2005/Atom'
       xmlns:gm='http://base.google.com/ns-metadata/1.0'>
  <id>
  http://www.google.com/base/feeds/itemtypes/en_US/jobs
  </id>
  <updated>2006-03-16T23:02:50.771Z</updated>
  <category scheme="http://base.google.com/categories/itemtypes" term="jobs"/>
  <title type="text">jobs</title>
  <content type="text">jobs</content>

  <link rel="related" type="application/atom+xml" href="http://www.google.com/base/feeds/snippets/-/jobs" title="Items in jobs"/>
  <link rel="self" type="application/atom+xml" href="http://www.google.com/base/feeds/itemtypes/en_US/jobs"/>
  <gm:item_type>jobs</gm:item_type>
    <gm:attributes>
       <gm:attribute name="job type" type="text"/>
       <gm:attribute name="job industry" type="text"/>
       <gm:attribute name="job function" type="text"/>
       <gm:attribute name="employer" type="text"/>
       <gm:attribute name="expiration date" type="dateTime"/>
       <gm:attribute name="pickup" type="boolean"/>
       <gm:attribute name="location" type="location"/>
    </gm:attributes>

</entry>
