Release Notes

The Google Base data API does not currently support digital upload of content, such as excel spreadsheets. You can use the image_link attribute to refer to an image that you are hosting on your site.

Some error messages are unclear. We are planning to improve these error messages for future releases. (523388 and 523673)

If you enter an extended ascii character, such as an ü character, it will be corrupted when it is stored. (210500)

New Features

The following new features were recently added to the Google Base data API.

JavaScript Tutorial: The new JavaScript Tutorial is a simple end-to-end example that presents all of the basic concepts of Google Base. It searches Google Base for vehicles, and then displays the results on a map. If you want to learn how to use the Google Base data API, this is a great place to start.

Crowding: You can use crowding to limit repetitive query results by placing a cap on similar items.

Adjusted names and attributes: In an effort to improve data quality, queries can now return results for exact attribute names and query terms, as well as synonyms of these.

Hiding items: You can use the app:draft extension element, defined in the Atom Publishing Protocol, to upload an item as a draft.

Troubleshooting: Disapproved items are now marked gm:disapproved.

Full-text queries: Google Base now supports the full functionality of the q query parameter as specified by GData.

Spell-checking: Full-text queries are now automatically spell-checked.
